vim-zsh-tmux-tmuxinator打造终极开发和写作环境

2015-08-09
技术

这篇文章讲的究竟是什么?

这是我这两天开发Rails时候使用的一些工具.经过简简单单的几个步骤,就可以打造出对程序员友好的开发环境.以及日常的写作环境.

很久以来,我们在编写文件的时候都会遇到这样的问题:

  • 由于编辑器的丑陋不够狂拽酷炫吊炸天导致的思维枯竭以及在妹子面前的编辑失常.
  • 上一条是我随便吹的
  • 编辑区域和shell命令不能完整的统一.以前的解决方案很简单,直接另开一个窗口,然后不断的alt+tab,终于找到终端,紧接着不断的cd cd cd 然后run一下命令,好.接着alt+tab好几遍才切换回需要编辑的文件,紧接着脑子一乱,妈蛋,我刚刚编辑文件以后的思路是什么来着?
  • 手贱关掉了终端.然后打开终端不断的cd cd cd vim 然后找到原来的编辑文件.

很显然,我们明明没有必要花时间在切换窗口上面.因为不断的切换,导致我们的思路容易中断.

再举个例子,我现在需要编辑rails项目,那么,在这个过程中我需要开6个终端,这6个终端有用来运行console的,有用来运行rails s的.有用来编辑项目本身的3个(三个比较方便,因为还有一个是gem,一个是todo),还有运行测试的终端.如果使用终端,那我就有两个选择.一是开6个tab,二是使用tmux开6个终端.但前者有一个非常大的缺陷,就是反应速度上较慢,每一个终端无法分屏,以及,问题最大的,初始化极其不方便.

很显然,这篇文章就是用来解决这种编辑时候的低效率问题.

也很显然,尽管我采用的工具都是尽量跟随大神的脚步,大神的配置,但依旧不能解决超级懒而且又想省时间的人.

从这篇文章,你大致可以学到什么?(前提是你需要稍微懂一点点Linux命令)

1.使用vim+zsh+tmux+tmuxinator来打磨自己的日常开发与写作工具.
2.我的使用流程以及几个配置.

vim + zsh + tmux + tmuxinator

  • vim 使用 spf13-vim
  • zsh 使用 oh my zsh
  • tmux 直接apt-get 一下就好.
  • tmuxinator 需要ruby环境.你可以直接安装ruby2.2,然后使用gem install 一下,不过我建议使用rvm安装ruby.

  • tmux状态栏(需要打字体补丁)
  • zsh 主题切换

涉及到的点有:

1.为什么要使用Web框架?
2.为什么要使用Rails框架?

1.为什么要使用Web框架?

很简单,追求开发效率.来个场景说明一下:

我登录我的新浪weibo,查看我的私信,浏览别人的webo,然后删除我觉得不好的内容.

那,这个过程的本质是什么呢? 就是取来我的weibo数据和别人的weibo数据,接着对数据进行增删改查.对吧?

其实你仔细想想,这个过程和你所了解的Web开发没有什么关系嘛!

如果世界上每一个人都是程序员,只要个终端链接数据库select一下取出数据不就好了?

聪明的你显然已经看出来了,可以这么做,但并不是每个人都是程序员啊!所以,Web开发的程序员就是将这个过程美化成用户友好的网站,屏蔽了客户端的复杂度,降低了用户的门槛.

技术

在茫茫人海中，为什么仅仅选择了Sublime作为主编辑器。

我的备选编辑器方案有四款。

  • vim 神之编辑器
  • emacs 编辑器之神
  • notepad++ 编辑器
  • 其他编辑器。

我的编辑器评判标准

语言方面要有HTML+CSS+JS+Python+autohotkey+markdown
兼顾可配置性和实用性。

其他编辑器有什么缺陷吗？
vim 符合终端时代的编辑模式，不符合GUI界面的编辑方法（可以键鼠操作），学习路径过于陡峭，插件配置过于恶心繁琐。
emacs 符合GUI界面的编辑方法，但是配置繁杂。
notepad++以前觉得非常方便，直到喜欢上了Sublime才知道，原来美观的东西也可以非常强大。
其他，使用的不多。

那么,这个Sublime究竟好在哪里？

  • 方便的Vim模式。
  • 各种语言特性支持。
  • 强大的社区支持。插件教程，balbalbala

如何优雅的使用Windows之文件以及文件夹规范

2015-02-24
技术

　系列之一：官方的基本快捷操作与你的文件分布应该是什么样子的。

首先，请反省一下自己。是不是桌面很乱，文件经常找不到？

其次，如果你使用Windows7以上的系统，反省一下，是不是经常每个盘根目录下面分类超过六个？

好，想到这里，我们就提出解决方法来。

我们把问题仔细分解一个一个来处理。

第一个，桌面乱，程序图标多

  1. 程序除非是绿色软件，或者开发环境，否则全部装在C盘的program fies里面。并且为软件创建快捷方式并命名为简写，比如xmind 简写为xm 为知笔记简写为wz ，新建一个文件夹，把快捷方式放到这里来。并且为这个文件夹配置好环境变量。我是将D:\Sysconfig存放在环境变量之中，这样就可以直接运行win+R 输入xm 回车，便可以打开xmind程序了。**这是为了把桌面上的无用的图标放在这里
  2. 这样的话，你就只需要管理那个文件夹就可以管理程序的运行了。

是不是很方便？

第二个,桌面乱，文件与文件夹多

1.这个好解决，推荐使用一款工具叫做Everything，这款软件只需要你好好的命名文件与文件夹，便可以秒搜。什么意思？比如我有一个文件 名字叫做 20150215如何优雅的使用windows之文件与文件夹规范，但是时间长了我忘了放在哪里，而且我只记得 2015年写的，关于文件夹的，因为写的是Blog，所以用的肯定是markdown文件，以md结尾,这个时候 我打开Everything，输入md 2015 文件，瞬间搜索到这个文件。Everything被称为文件搜索类程序中的小李飞刀。天下武功，无坚不摧，唯快不破。就是这个道理。你不需要知道文件在哪，你只要记得文件怎么命名的就好。只是这个软件对使用者有一定的命名意识。一定要有一定的命名规范。

2.好好分类，不要随手建立一个新建文件夹或者随手建立一个新建文件然后连名字都不写。在几个小时后你就没有办法找到你要的文件了。这个时候你去一个一个打开看看？

第三个,文档怎么分类命名？

1.我认为，对于文档，文件名中应该包含 时间+文件标题或者几个字概括内容+是否完成。比如 20150215记录文件夹灵感Undo.md 比如 20150215常州恐龙园门前合照.jpg 或者没必要命名时间的一些资源文件：AndroidButtonIOS风格.psd
2.对于文件夹，我个人倾向于使用英语。比如File Document Workspace ;我感觉我的文件可能不算非常的多，所以文件夹的命名并没有什么特点。比如我的正在编写的Android代码就在D:\Workspace\Android 里面。而绝大部分对排版要求不是非常高的我都是存放在为知笔记里面。

下面是我的关于分区的看法，我们将硬盘分为四个区.

  • C盘安装一些必须运行的环境,比如JDK 微软的运行库 补丁还有一些其他的非绿色的软件.我单独给了一个根文件夹用于存放安装的环境。叫做DevEnv
  • D盘安放一些IDE程序,编写的Office文档以及暂时的电子书文件。
  • E盘安放一些用来休闲娱乐和已经保存下来的资料,游戏，以及下载区和云同步的文件。
  • F盘安放影视文件，以及私人的图片文件。

这样分区是为了更好的区分自己的状态,不要在学习的时候去其他的盘符里面.

配置以后的快捷操作Win+R

前面提到过一个Win+R。
这里附加一个链接.以供来者学习
最绿色最高效，用win+r启动常用程序和文档
我将将绝大部分的程序快捷方式放到了sysconfig,今后我们还会将这个文件夹

回顾一下就是,

  1. 按照工作还是娱乐状态把文件放在DE盘里.
  2. 在盘的内部按照种类分别存放
  3. 在工作盘里面配置一个大量快捷方式的文件加并把路径加到path中
  4. 在工作盘内部增加一个inbox用于存放临时的文件.把经常需要编辑的东西放到里面.
